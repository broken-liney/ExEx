# Full Stack Development

## What is Full Stack?
**Full Stack Development** refers to building and maintaining **both the front-end and the back-end** of an application
means you develop the the site's design and functionality

A **Full Stack Developer** understands:
- how the entire system works
- what the user sees
- how data is processed and stored behind the scenes

---

## Front-End (Client Side)
The **front-end** is everything the user interacts with directly

### Common Front-End Language:
- **HTML** – structure of the page
- **CSS** – styling and layout
- **JavaScript** – interactivity and logic
- Frameworks/Libraries:
  - React
  - Vue
  - Angular
  - and more :p

### Responsibilities:
- User interface (UI)
- User experience (UX)
- Responsive design
- Browser compatibility
- And to flexing that you can make a website XD

---

## Back-End (Server Side)
The **back-end** handles logic, databases, and server operations.

### Common Back-End Languages:
  - JavaScript (Node.js)
  - Python
  - PHP
  - Java
- **Frameworks**:
  - Express
  - Django
  - Laravel
- **Databases**:
  - MySQL
  - PostgreSQL
  - MongoDB

### Responsibilities:
- Business logic
- Authentication & authorization
- Database management
- APIs

---

## APIs & Communication
Front-end and back-end communicate through **APIs** (usually REST or GraphQL).

- Sends requests (GET, POST, PUT, DELETE)
- Receives and processes responses (JSON)

---

## DevOps & Tools
A Full Stack Developer often knows basic deployment and tooling.

### Examples:
- Git & GitHub
- Docker (basic understanding)
- CI/CD
- Cloud services
- Linux basics

---

## At The End
A **Full Stack Developer** is not someone who knows *everything* in programming,  
but someone who understands **how parts of web connect together**! cya!
